# なぜHe初期化を疑ったのか

He初期化は

[
W\sim N(0,\sqrt{2/fan_{in}})
]

です。

つまり

**fan_inを少し間違えるだけで**

```

↓

Conv出力
↓

次のConv入力
↓

次のConv出力
```

と何層も影響します。

これは

ドミノ倒しのようなものです。

```
最初のドミノ
↓

少ししか倒れない
↓

次ももっと小さい

↓

さらに小さい
```

結果として

```

↓

0.004

↓

0.0007
```

のようになります。

今回観測した値と非常によく一致していました。

## 現在、私が最も疑っているもの

ここまで調査した結果、優先順位は次のように変わっています。

1. **Convolution.backward() の実装**
2. **Flatten〜Affineへ渡る入力スケール**
3. **Optimizer（学習率・更新処理）**
4. He初期化（現在は優先度が下がりました）

つまり、**He初期化は「有力な仮説」だったため詳しく調査しましたが、コードを確認した結果、その仮説はかなり弱くなりました。**

このように一つずつ可能性を潰していくのが、ディープラーニング実装のデバッグでは非常に重要な進め方です。
