賛成です。

ここまでで**Recorder・可視化基盤**については、実装できるレベルまで仕様が固まりました。以下に、実装仕様として整理します。

---

# 1. プロジェクトの目的

## 最終目標

AIの内部動作を理解するための**汎用的な深層学習可視化フレームワーク**を構築する。

## 第一段階

* VGG16をフルスクラッチ実装
* 学習・推論
* Forward/Backwardの理解
* 特徴マップの可視化

## 設計方針

* AIの理解を最優先
* 処理の見える化
* 責務を明確に分離
* 将来的にResNetなどへ拡張可能

---

# 2. システム全体

```text
Trainer
    │
    ▼
Model.forward()
    │
    ▼
HookManager
    │
    ▼
Recorder
    │
    ▼
ForwardRecord(.pt)
    │
──────────────────────────
    （後処理）
──────────────────────────
    │
    ▼
ImageRenderer
    │
    ▼
Viewer
├── OverviewViewer
└── FeatureMapViewer
```

---

# 3. HookManager

## 役割

各レイヤーのForwardを監視する。

取得する情報

* input_tensor
* output_tensor

Recorderへ渡す。

---

# 4. Recorder

## 役割

Forward中に取得した情報をまとめて保存する。

Recorderは

* 描画しない
* 学習制御しない
* テンソルを加工しない

保存だけ担当する。

---

# 5. 保存タイミング

## 学習

YAMLで指定

例

```yaml
visualizer:
  train:
    enabled: true
    every_epoch: 5
```

なら

```text
Epoch5
Epoch10
Epoch15
```

のみ保存。

---

## 推論

毎回保存。

---

# 6. 保存対象

## 保存するレイヤー

**すべて**

例

* Conv
* ReLU
* MaxPool
* Flatten
* Affine
* Dropout
* Softmax

---

## 保存するサンプル

バッチ内の**1サンプルのみ**

例

```yaml
visualizer:
  train:
    sample_index: 0
```

どのサンプルを保存するかは **Trainer** が決定する。

Recorderは渡されたサンプルを保存するだけ。

---

# 7. 保存形式

```
.pt
```

`torch.save()` を利用。

---

# 8. 保存単位

## 1 Forward = 1ファイル

例

```
epoch001_batch010_sample000.pt
```

# 9. ForwardRecord

```
ForwardRecord
│
├── metadata
│
└── layers
    ├── LayerRecord
    ├── LayerRecord
    └── ...
```

---

## metadata

現在決まっているもの

* mode
* epoch
* batch
* sample_index
* label
* prediction

※ 必要に応じて今後追加可能。

---

# 10. LayerRecord

```
LayerRecord
├── layer_name
├── layer_type
├── input_shape
├── output_shape
├── input_tensor
└── output_tensor
```

保存しないもの

* weight
* bias
* kernel_size
* stride
* padding

---

# 11. ImageRenderer

## 役割

テンソルを画像へ変換する。

例

```
Tensor
    │
    ▼
ImageRenderer
    │
    ▼
Image
```

描画レイアウトは担当しない。

---

# 12. Viewer

Viewerは画像を並べるだけ。

---

## OverviewViewer

用途

ネットワーク全体を俯瞰する。

表示順

Forward実行順

```
Input
↓
Conv1
↓
ReLU1
↓
Pool1
↓
...
```

画像

| チャンネル数 | 表示                |
| ------ | ----------------- |
| 1      | グレースケール           |
| 3      | RGB               |
| 4以上    | チャネル平均を取ったグレースケール |

1レイヤーにつき1枚表示。

---

## FeatureMapViewer

用途

レイヤー内部を解析する。

1レイヤーにつき1Figure。

例

```
Conv3

□ □ □ □ □ □ □ □
□ □ □ □ □ □ □ □
□ □ □ □ □ □ □ □
...
```

全チャネルを表示する。

ページ分割しない。

---

# 13. 学習と可視化の分離

Forward中

```
Forward
    │
Recorder
    │
ForwardRecord保存
```

後処理

```
ForwardRecord
    │
ImageRenderer
    │
Viewer
```

学習中に画像生成は行わない。

---

# 14. 責務一覧

| クラス              | 責務                    |
| ---------------- | --------------------- |
| Trainer          | 学習制御・保存タイミング・保存サンプル選択 |
| Model            | 順伝搬・逆伝搬               |
| HookManager      | レイヤーの入出力取得            |
| Recorder         | ForwardRecord生成・保存    |
| ForwardRecord    | 1回のForward結果保持        |
| LayerRecord      | レイヤーごとの入出力保持          |
| ImageRenderer    | テンソル→画像変換             |
| OverviewViewer   | 全レイヤーを一覧表示            |
| FeatureMapViewer | 全チャネル表示               |

---

## 私から1つだけ追加したい仕様

実装を始める前に、**ForwardRecord・LayerRecord・Recorder・ImageRenderer・Viewer のクラス図**を作成することをおすすめします。

ここまで仕様が固まっているので、クラス図を作ることで、

* クラス間の依存関係
* インターフェース
* メソッド
* 保持する属性

まで明確になり、実装中に設計がぶれにくくなります。

私は**次のステップはクラス設計（クラス図・メソッド設計）**に進むのが最も効率的だと考えています。
